# Changelog

All notable changes to this project will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/), and this project adheres to [Semantic Versioning](https://semver.org/).

## [1.0.0] — 2026-04-09

### Added

**01 — Foundation**

- README, CONTRIBUTING, CODE_OF_CONDUCT, SECURITY, CHANGELOG templates
- License selection guide
- GitHub issue templates (bug, feature request, improvement, new profile)
- GitHub PR template

**02 — Architecture**

- Architecture Decision Record (ADR) template and guide
- Coding standards template
- Tech stack profile template

**03 — Git Workflow**

- Branching strategy guide (trunk-based, GitHub Flow, Gitflow)
- Commit conventions (Conventional Commits)
- Code review checklist
- Merge policy template

**04 — Quality**

- Release audit framework: 12 weighted checklists with GO/NO-GO scoring
- Audit checklists: Clean Architecture, DDD, Clean Code, KISS, DRY, SOLID, Decoupling, Security, Tests, CI/Build, Documentation, Performance
- Test strategy template
- Bug report template
- E2E test status template
- Executive quality summary template

**05 — Release**

- Release process checklist
- Semantic versioning guide
- Deployment checklist
- Rollback playbook

**06 — Community**

- Contributor onboarding guide
- Governance model (BDFL to steering committee)
- Issue triage process with labels and SLAs
- Public roadmap template
- Communication plan template

**07 — Operations**

- Incident response playbook with post-mortem template
- Security vulnerability response process
- Monitoring checklist (four golden signals)

**Profiles**

- FastAPI + Vue 3 (profile + commands)
- Django + React (profile + commands)
- Spring Boot + Angular (profile + commands)

**Examples**

- Sample audit report (GO CONDITIONAL scenario)
- Sample ADR (PostgreSQL selection)

**Infrastructure**

- Markdown linting via GitHub Actions CI
